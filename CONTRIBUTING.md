# 贡献指南（共建）

感谢参与 structdesign 共建！本项目目标是做一套**算得准、用得起、看得懂**的开源结构设计软件。

## 三条开发铁律

1. **求解方法优先级**：规范公式 → 无依据处用有限元 → 有依据的工程方法。每个数都要有可引用依据。
2. **凡决策必留痕**：用 `trace.py` 记录 条文/公式/代入/结果，供计算书复核。
3. **验证驱动**：每个功能必须有**解析解或守恒律**测试（如 `PH³/3EI`、`wL²/8`、欧拉 `Pcr`、荷载守恒、平行轴定理、规范表锚点……）。**先想怎么验证 → 写测试 → 写实现。**

## 测试约定

- 无需 pytest：每个 `tests/test_*.py` 是可独立运行的脚本，文件顶部 docstring 写独立手算过程，
  `__main__` 内逐个跑 `test_*` 打印 PASS/FAIL 并以非 0 退出码表示失败。
- 提交前请全套通过：

  ```bash
  for t in tests/test_*.py; do python "$t"; done    # 应全绿
  ```

## 环境

- 建议 Python 3.9；依赖见 `requirements.txt`（numpy / matplotlib / PyQt5 / ezdxf / plotly）。
- 可选：`anthropic`（AI 助手 LLM 模式）、`pandoc`（计算书转 Word）、ODA File Converter（导入 .dwg）。

## 提交流程

1. Fork → 新建分支 → 编码（遵守上面三条铁律）。
2. 为新功能补解析解/守恒律测试，确保全套绿。
3. 提交 PR，说明：解决什么问题、依据哪条规范、如何验证。

## 诚实边界

请勿让"未实现/简化"的功能在计算或图面上**假装精确**。简化必须在计算书与代码注释中如实标注。
